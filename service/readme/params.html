Main parameters

Parameter Type Req Descriptions
apiURL string

The api base url

Default: https://www.voicebase.com/services

Example: 
apiURL: 'https://beta.voicebase.com/services'
apiVersion float

The api version

Default: 1.1

Example: 
apiVersion: 1.1
apiKey string

your voicebase api key

Example: 
apiKey: 'ABCDEFGH-1234-ABCD-1234-1A2B3C4D5F6G'
password string

your voicebase api password

Example: 
password: 'you_vb_password'
token string

is indicated instead of apikey and password

Example: 
token: 'db250aaf...e581a'
tokenTimeOut integer

Live Timeout for token

Example: 
tokenTimeOut: 15
mediaID string

mediafile id

Example: 
mediaID: 'ABCD1234DCBA'
externalId string

is indicated instead of MediaID

Example: 
externalId: 'my-media-file'
playerType string

Player type. Supported players list here

Example: 
playerType: 'jwplayer'
playerId string

Player ID

Example: 
playerId: 'playerBlock'

Additional parameters

Parameter Type Descriptions
mediumResponsiveWithSpeakers int

Width for adding responsive class if Speakers are defined

Default: 600

Example: 
mediumResponsiveWithSpeakers: 580
mediumResponsive int

Width for adding responsive class

Default: 450

Example: 
mediumResponsive: 480
minResponsive int

Width for adding responsive small class

Default: 400

Example: 
minResponsive: 580
showMore boolean

Add Show More button for transcript and keywords sections

Default: false

Example: 
showMore: true
animation boolean

Show animation spinner in timeline when loading or waiting for response

Default: true

Example: 
animation: false
contextMenu boolean

Show context menu for copying and sharing URL of keyword or position

Default: true

Example: 
contextMenu: false
actionflag object

Interception of default events of buttons

See list of webhooks here

Example: 
actionflag: {
    remove: false,
    comment: false
}
webHooks mixed

Interception of default events of buttons

Default: false

See list of webhooks here

Example: 
webHooks: {
    socialShare: function(){
        myCustomJSfunction();
    }
}
trackEvents boolean

Allows to trace Play, Pause, Seek events, click on keywords and transcript text, and gather GA statistics

Require Google Analytics

Default: false

Example: 
trackEvents: true
stream mixed

to use the streaming for media-file (http (html5) | rtmp (flash) | true (html5, http))

Only in JW Player

Default: false

Example: 
stream: "rtmp"
example boolean

Used only in VoiceBase example files

Default: false

Example: 
example: true
exampleTokenUrl string

Used only in VoiceBase example files

Default: true

Example: 
exampleTokenUrl: "http://exmaple.com/exampleToken"
toggleBlocks boolean

If false, then keywords, media, transcript and comments blocks are always opened and can't be collapsed

Default: true

Example: 
toggleBlocks: false
searchBarBlock string

ID of the alternative block for showing the search bar

Default: 'vbs-searchbar-block'

Example: 
searchBarBlock: 'id-block'
searchBarBlockWidth integer or false

Custom width for search bar block

Default: false

Example: 
searchBarBlockWidth: 58
searchBarOuter boolean

Move search bar out of keyword section

Default: true

Example: 
searchBarOuter: false
tabView boolean

Keyword, transcript and comments sections are finding in tabs

If tabView is equal true, then parameters

  • searchBarOuter
  • keywordsBlock
  • keywordsHeight
  • keywordsWidth
  • transcriptBlock
  • transcriptHeight
  • transcriptWidth
  • commentsBlock
  • commentsWidth

will have default values

Default: false

Example: 
tabView: false
debug boolean

If debug is equal true, then after entering ctrl+alt+g, logs will go to the console of browser, loggly.com and screen

Information of logs: browser info, stream, mediaId, mode (flash or http), status of comments, transcripts, keywords, metadata; url of stream, url of site and player type

Default: true

Example: 
debug: false
mediaTypeOverride string

Can be: 'video' or 'aduio'

This parameter override type of media file coming from metadata.

Need when users index audio but stream video via remote streams or their own streaming server.

Default: null

Example: 
mediaTypeOverride: 'video'
localApp boolean

This parameter means that this is a local application without requests to server.

Transcript and keywords are created from YouTube subtitles.

This work is only for YouTube player.

Default: false

Example: 
localApp: true
markersInNativeTimeline boolean

Keywords, markers are shown in native timeline of kaltura

This is work only for kaltura player.

Default: false

Example: 
markersInNativeTimeline: true
modalErrors boolean

Error messages are shown in modal popups or inline in interface

Default: false

Example: 
modalErrors: true
modalSave boolean

Messages of saving transcript are shown in modal popups or inline in interface

Default: false

Example: 
modalSave: true

Share parameters

Parameter Type Descriptions
shareUrl string

Custom url for sharing without parameters

Example: 
shareUrl: "http://some_url.com/"
shareParams object

Custom parameters that will be added to URL with Sharing

Example: 
shareParams: {"param1": "value1"; "param2": 12}
shareTitle string

Title shown in Share dialog

Default: "Checkout this!"

Example: 
shareTitle: "Follow me!"
addThisButtons array

Enumeration of addThis buttons in Share dialog

Default: ['gmail', 'facebook', 'twitter', 'google_plusone_share', 'compact']

See more buttons in addThis site

Example: 
addThisButtons: ['facebook', 'compact']
voicebaseShare boolean

Add button "Share with E-mail" in Share Dialog

Default: false

Example: 
voicebaseShare: true
zeroclipboard string

Path to swf file of zeroClipboard plugin for Copy URL to Clipboard

Default: "ZeroClipboard.swf"

Example: 
zeroclipboard: "http://ex.co/zc.swf"
...
zeroclipboard: "/ZeroClipboard.swf"

Transcript Block parameters

Parameter Type Descriptions

Custom height for transcript block

Default: 148

Example:

Custom width for transcript block

Default: player or content width

Example: 
transcriptWidth: 580
transcriptResizable boolean

Turning on the possibility to change the transcript block size

Default: false

Require jQuery UI

Example: 
transcriptResizable: true
transcriptHighlight integer

Interval of the transcript highlight time (in seconds). Minimum 5 (if less than 5, the default is 5, if 0 - forbids highlight and scrolling

Default: 5

Example: 
transcriptHighlight: 15
filterSpeaker string

Initialize filter keywords by speakers. By default keywords are shown for all speakers.

Default: 'all'

Example: 
filterSpeaker: 'John'
turnTimes boolean

If speaker turn info is available in the transcript, show the time each turn starts in the transcript section

Default: false

Example: 
turnTimes: true
lineBreak float

Determines the period of time for word transferring to a new line in transcript

Default: false

Example: 
lineBreak: 1.3
transcriptCheckTimer Integer

Time interval for checking of saving transcript

Default: 10

Example: 
transcriptCheckTimer: 5
humanOnly boolean

Shows the transcripts recognized by the machine and human. true - shows the transcript recognized only by human

Default: false

Example: 
humanOnly: true
